哈希表 2024.2.3 leetcode 1 两数之和 题目直达：https://leetcode.cn/problems/two-sum/description/?envType=study-plan-v2&envId=top-100-liked 思路 最开始的思路直观用for循环去检查对于一个数字n，在数组中有没有target - n的数字存在，如果存在则表明在这个数组中有两个数字之和为target，目的达成。 这个思路时间复杂度最差为n^2吧，因为需要遍历两次，空间复杂度倒不会很高。但是看一下数组的长度，在2 <= nums.length <= 10^4这个范围，感觉似乎是可以的？ 但是其中也是可以有优化的地方的，比如对于数字n来说，想在整个数组中找到target - n可以用hash表来实现；当每出现一个未曾出现的数字就把它添加到hash表中，然后在hash表中寻找target - n速度就很快，时间复杂度为O(1)。 为什么hash表的时间复杂度是O(1)呢？ 使用空间换时间嘛，他的空间复杂度是O(n)。hash表把键扔到hash函数中进行计算，然后直接访问数组中这个下标的位置，就能做到O(1)的时间复杂度。 所以采用哈希表的方法，就可以做到O(n)的时间复杂度，一边把未命中的数字放入到hash表中，查到命中的数字就结束。 代码如下： // leetcode 1. 两数之和 package main import "fmt" func main() { // 定义测试用例 nums := []int{2, 7, 11, 15} target := 9 fmt.Println(twoSum(nums, target)) } func twoSum(nums []int, target int) []int { // 定义哈希表 hashMap := make(map[int]int) // 遍历数组 for i, num := range nums { if j, ok := hashMap[target-num]; ok { return []int{j, i} } // 将当前元素存入哈希表，键是num，值是i hashMap[num] = i } return nil } go语法复习 变量定义 首先是变量的定义，在函数内部就采用a := 1这样的形式就可以自动判别类型；对于数组的定义也是相同的操作nums "= []int{2,7,11,15}"...
