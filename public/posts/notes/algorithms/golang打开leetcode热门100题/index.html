leetcode 1 两数之和#

题目直达：https://leetcode.cn/problems/two-sum/description/?envType=study-plan-v2&envId=top-100-liked

思路#

最开始的思路直观用for循环去检查对于一个数字n，在数组中有没有target - n的数字存在，如果存在则表明在这个数组中有两个数字之和为target，目的达成。

这个思路时间复杂度最差为n^2吧，因为需要遍历两次，空间复杂度倒不会很高。但是看一下数组的长度，在2 <= nums.length <= 10^4这个范围，感觉似乎是可以的？

但是其中也是可以有优化的地方的，比如对于数字n来说，想在整个数组中找到target - n可以用hash表来实现；当每出现一个未曾出现的数字就把它添加到hash表中，然后在hash表中寻找target - n速度就很快，时间复杂度为O(1)。

为什么hash表的时间复杂度是O(1)呢？

使用空间换时间嘛，他的空间复杂度是O(n)。hash表把键扔到hash函数中进行计算，然后直接访问数组中这个下标的位置，就能做到O(1)的时间复杂度。

所以采用哈希表的方法，就可以做到O(n)的时间复杂度，一边把未命中的数字放入到hash表中，查到命中的数字就结束。

代码如下：

// leetcode 1. 两数之和
package main

import "fmt"

func main() {
	// 定义测试用例
	nums := []int{2, 7, 11, 15}
	target := 9
	fmt.Println(twoSum(nums, target))
}

func twoSum(nums []int, target int) []int {
	// 定义哈希表
	hashMap := make(map[int]int)
	// 遍历数组
	for i, num := range nums {
		if j, ok := hashMap[target-num]; ok {
			return []int{j, i}
		}
		// 将当前元素存入哈希表，键是num，值是i
		hashMap[num] = i
	}
	return nil
}

if语法#

在 Go 语言中，分号 ; 用于分隔语句。在这个 if 语句中，它用于将初始化语句和条件表达式分开。

if 语句的完整形式是 if initialization; condition { }。初始化语句（j, ok := hashMap[target-num]）在条件表达式（ok）之前执行，并且初始化语句中声明的变量（在这里是 j 和 ok）在 if 语句的块中都是可见的。

所以，if j, ok := hashMap[target-num]; ok { return []int{j, i} } 的意思是：

  1. 尝试从 hashMap 中获取键为 target-num 的值，并将结果存储在 j 和 ok 中。
  2. 如果 ok 为 true（也就是说，hashMap 中存在键为 target-num 的项），那么执行 if 语句的块，返回一个包含 j 和 i 的切片。

make用法#

在 Go 语言中，make 函数用于初始化内建的数据类型，包括切片（slice）、映射（map）和通道（chan）。make 函数返回的是这些类型的初始化值（非零值），而不是指针。

以下是 make 函数的使用方法：

  1. 初始化切片：s := make([]int, 10) 创建一个长度和容量都是 10 的整数切片。你也可以指定切片的容量：s := make([]int, 10, 20) 创建一个长度是 10，容量是 20 的整数切片。

  2. 初始化映射：m := make(map[string]int) 创建一个空的字符串到整数的映射。

  3. 初始化通道：c := make(chan int) 创建一个无缓冲的整数通道。你也可以创建一个有缓冲的通道：c := make(chan int, 10) 创建一个缓冲大小为 10 的整数通道。

注意，make 函数只能用于切片、映射和通道，不能用于数组、结构体等其他类型。对于这些类型，你可以使用 new 函数或字面量来创建和初始化。

leetcode 49 字母异或词分组#

题目链接：https://leetcode.cn/problems/group-anagrams/description/?envType=study-plan-v2&envId=top-100-liked

// leetcode 49 字母异位词分组
// 使用排序的思想，把每个单词排序后的作为key，原单词作为value，存入map中
func groupAnagrams(strs []string) [][]string {
	hashmap := map[string][]string{}
	for _, str := range strs {
		s := []byte(str) // 转换成byte数组，然后排序
		// sort.Slice()函数的第二个参数是一个匿名函数，用来定义排序规则
		sort.Slice(s, func(i, j int) bool { return s[i] < s[j] })
		key := string(s)
		// append函数需要使用他的返回值来更新切片，接受两个参数，第一个是切片，第二个是要添加的元素
		hashmap[key] = append(hashmap[key], str)
	}
	res := [][]string{} // res是一个二维切片
	// 遍历的时候v代表value: []string，所以就在下直接添加到二维切片中了
	for _, v := range hashmap {
		res = append(res, v)
	}
	return res
}

leetcode 128 最长连续序列#

没看题解自己写的，有点费内存，总体思路就是如果num是一个连续序列的下一个，那么他的前一个在hash表中一定有非0值。

因为是利用前一个数字的hashmap的key来更新的，所以不用考虑去重的问题。

// leetcode 128 最长连续序列
func longestConsecutive(nums []int) int {
	sort.Ints(nums)
	hashmap := make(map[int]int)
	res := 0
	// 对于一个数num，如果和前面是连续序列，那么hashmap[num-1]一定不为0
	for _, num := range nums {

		if hashmap[num-1] == 0 {
			hashmap[num] = 1
		} else {
			hashmap[num] = hashmap[num-1] + 1
		}

		if hashmap[num] > res {
			res = hashmap[num]
		}
	}
	return res
}

leetcode 283 移动零#

// leetcode 283	移动零
func moveZeroes(nums []int) {
	// 双指针，一个指针指向左边已经排好的最后一个
	// 另外一个指针指向还没有排好的元素，并且一直向右移动，遇到非零数就交换左右指针，然后左边的指针向右边移动一个位置
	left, right := 0, 0
	for right < len(nums) {
		if nums[right] != 0 {
			nums[left], nums[right] = nums[right], nums[left]
			left++
		}
		right++
	}
}

go中交换两个变量的值#

left 和 right 并不是指针，而是索引。它们是用来访问 nums 切片中的元素的。

nums[left], nums[right] = nums[right], nums[left] 这行代码是在交换 nums 切片中索引为 left 和 right 的两个元素。在 Go 语言中，你可以使用这种简洁的语法来交换两个变量的值，而不需要引入一个临时变量。

这行代码的工作原理是：首先，Go 语言会计算等号右边的两个表达式 nums[right] 和 nums[left]，然后将这两个值赋给等号左边的两个变量 nums[left] 和 nums[right]。因为等号右边的表达式在赋值之前就已经被计算了，所以这个操作可以正确地交换两个变量的值。

go中指针#

在 Go 语言中，你可以使用 * 符号来定义一个指针类型的变量，使用 & 符号来获取一个变量的内存地址。

以下是一个定义和使用指针的例子：

var x int = 1
var p *int = &x // p 是一个指向 int 类型变量的指针

fmt.Println(*p) // 输出 1，*p 是 p 指向的变量的值

*p = 2 // 改变 p 指向的变量的值
fmt.Println(x) // 输出 2，x 的值已经被改变

在这个例子中，p 是一个指向 int 类型变量的指针，&x 是变量 x 的内存地址。*p 是 p 指向的变量的值，所以你可以通过 *p 来获取或改变 p 指向的变量的值。

需要注意的是，当你试图获取一个变量的内存地址时，该变量必须已经被定义。如果你试图获取一个未定义的变量的内存地址，Go 语言会报错。同样，当你试图通过一个指针来获取或改变一个变量的值时，该指针必须已经被初始化，也就是说，它必须指向一个有效的内存地址。如果一个指针没有被初始化，它的值是 nil，试图通过一个 nil 指针来获取或改变一个变量的值会导致运行时错误。

leetcode 11 盛最多水的容器#

// leetcode 71	盛最多水的容器
func maxArea(height []int) int {
	left, right := 0, len(height)-1
	// 注意先对res先初始化
	res := min(height[left], height[right]) * (right - left)
	for left < right {
		// 每次把height[left]和height[right]中最小的那个向中间移动
		// 移动最小的面积不一定会增大，移动最大的面积一定会减小
		// 用res来记录遇到的最大值
		if height[left] < height[right] {
			left++
		} else {
			right--
		}

		s := min(height[left], height[right]) * (right - left)
		if res < s {
			res = s
		}
	}
	return res
}

leetcode 15 三数之和#

// leetcode 15	三数之和
func threeSum(nums []int) [][]int {
	// 先对数组进行排序
	sort.Ints(nums)
	res := [][]int{}
	hashtable := make(map[[3]int]int)
	for i := 0; i < len(nums)-2; i++ {

		if i != 0 && nums[i] == nums[i-1] {
			continue
		}

		left := i + 1
		right := len(nums) - 1

		for left < right {
			sum := nums[i] + nums[left] + nums[right]

			if sum == 0 {
				if hashtable[[3]int{nums[i], nums[left], nums[right]}] == 0 {
					res = append(res, []int{nums[i], nums[left], nums[right]})
					hashtable[[3]int{nums[i], nums[left], nums[right]}]++
				}
				left++
				right--
			} else if sum < 0 {
				left++
			} else {
				right--
			}
		}
	}
	return res
}

leetcode 42 接雨水#

func trap_copilot(height []int) int {
	stack := []int{} // 存储索引
	res := 0
	for i := 0; i < len(height); i++ {
		for len(stack) != 0 && height[i] > height[stack[len(stack)-1]] {
			top := stack[len(stack)-1]   // 获取栈顶元素的索引
			stack = stack[:len(stack)-1] // 弹出栈顶元素
			if len(stack) == 0 {
				break
			}
			distance := i - stack[len(stack)-1] - 1                                    // 计算宽度
			boundedHeight := min(height[stack[len(stack)-1]], height[i]) - height[top] // 计算高度
			res += distance * boundedHeight                                            // 计算体积并添加到 res 中
		}
		stack = append(stack, i) // 将 i 压入栈中
	}
	return res
}

leetcode 3 无重复字符的最长子串#

// leetcode 3	无重复字符的最长子串
func lengthOfLongestSubstring(s string) int {

	var queue string
	maxlen := 0
	for i := 0; i < len(s); i++ {
		// 如果队列中没有相同的字母，则加入该字母；否则队列向右边滑动窗口，直到没有重复的为止
		if !strings.Contains(queue, string(s[i])) {
			queue += string(s[i])
			if len(queue) > maxlen {
				maxlen = len(queue)
			}
		} else {
			// 滑动窗口前记得把最新的字符串加入
			queue += string(s[i])
			// 这里注意比对重复的，只用观察之前的就可以了，不要把新加入的字母也纳入比较
			for strings.Contains(queue[:len(queue)-1], string(s[i])) {
				queue = queue[1:]
			}
		}
	}
	return maxlen
}

func findAnagrams(s string, p string) []int {
	res := []int{}
	var queue string
	for i := 0; i <= len(s)-len(p); i++ {
		// 窗口固定大小为len(p)
		// 如果该字符往后数len(p)个字符的字符串不是异位词，则窗口右移一个单位
		queue = s[i : i+len(p)]
		if isAnagram(queue, p) {
			res = append(res, i)
		}
	}
	return res
}
func isAnagram(queue string, p string) bool {
	if len(queue) != len(p) {
		return false
	}
	count := [26]int{}
	for i := 0; i < len(queue); i++ {
		count[queue[i]-'a']++
	}
	for i := 0; i < len(p); i++ {
		count[p[i]-'a']--
	}
	for _, c := range count {
		if c != 0 {
			return false
		}
	}
	return true
}

leetcode 560 和为K的子数组#

前缀和#

前缀和的计算，然后两个前缀和相减就是一个子串的和，再与k进行比较是否相等

func subarraySum1(nums []int, k int) int {
	res := 0
	// 计算前缀和
	prefixSum := make([]int, len(nums))
	// 外层循环控制prefixSum的存储；内存循环控制前缀和的计算
	for i := 0; i < len(nums); i++ {
		count := 0
		for j := 0; j < i; j++ {
			count += nums[j]
		}
		count += nums[i]
		prefixSum[i] = count
	}

	// 看两个前缀和的差是否会等于k
	for i := 0; i < len(prefixSum); i++ {
		for j := i; j < len(prefixSum); j++ {
			var dif int
			if i == 0 {
				dif = prefixSum[j]
			} else {
				dif = prefixSum[j] - prefixSum[i-1]
			}
			if dif == k {
				res++
			}
		}
	}
	return res
}

func subarraySum2(nums []int, k int) int {
	res := 0
	hashmap := map[int]int{0: 1} //	初始化键值0:1
	presum := 0
	// 由于使用hash表进行存储，所以只用O(n)的时间复杂度计算前缀和即可
	for i := 0; i < len(nums); i++ {
		presum += nums[i]
		if hashmap[presum-k] > 0 {
			res += hashmap[presum-k]
		}
		hashmap[presum]++
	}
	return res
}

leetcode 239 滑动窗口最大值#

// leetcode 239 滑动窗口最大值
func maxSlidingWindow(nums []int, k int) []int {
	var res []int
	queue := []int{}
	// queue初始化
	for i := 0; i < min(k, len(nums)); i++ {
		queue = append(queue, nums[i])
	}

	// 这里i大于等于len(nums)，是为了执行最后一个元素的判断
	for i := k; i <= len(nums); i++ {
		// 获取队列中的最大值
		maxium := queue[0]
		for j := 0; j < len(queue); j++ {
			if maxium < queue[j] {
				maxium = queue[j]
			}
		}
		res = append(res, maxium)
		//	当在合法范围内才能移动窗口
		if i < len(nums) {
			queue = queue[1:]
			queue = append(queue, nums[i])
		}
	}
	return res
}

单调队列#

// leetcode 239 滑动窗口最大值
// 单调队列
type MotonicQueue struct {
	queue []int
}

func NewMotonicQueue() *MotonicQueue {
	return &MotonicQueue{
		queue: make([]int, 0),
	}
}

func (m *MotonicQueue) Front() int {
	return m.queue[0]
}

func (m *MotonicQueue) Back() int {
	return m.queue[len(m.queue)-1]
}

func (m *MotonicQueue) Empty() bool {
	return len(m.queue) == 0
}

func (m *MotonicQueue) Push(val int) {
	for !m.Empty() && val > m.Back() {
		m.queue = m.queue[:len(m.queue)-1]
	}
	m.queue = append(m.queue, val)
}

func (m *MotonicQueue) Pop(val int) {
	if !m.Empty() && val == m.Front() {
		m.queue = m.queue[1:]
	}
}

func maxSlidingWindow(nums []int, k int) []int {
	var res []int
	queue := NewMotonicQueue()
	length := len(nums)

	// 先将前面k个元素放入队列
	for i := 0; i < k; i++ {
		queue.Push(nums[i])
	}
	// 记录前面k个元素的最大值
	res = append(res, queue.Front())
	//	开始滑动窗口
	for i := k; i < length; i++ {
		// 滑动窗口移除最前面的元素
		queue.Pop(nums[i-k])
		// 滑动窗口添加最后面的元素
		queue.Push(nums[i])
		// 记录最大值
		res = append(res, queue.Front())
	}
	return res
}

push的时候，如果比前面的数字大，则需要删除前面小的数字；否则直接加到最后

pop的时候，除非是出去的元素是最大值，否则就不用更新
