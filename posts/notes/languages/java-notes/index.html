一、IDEA使用#

1、运行和调试#

运行Shift+F10：直接运行程序得到结果，遇到断点不会停止哦

调试Shift+F9：

遇到断点会停止执行，并给出此时变量的值，在调试过程中如果是按下Shift+F9/F10，是让程序重新运行；

按下F9是让程序继续运行，直到下一个断点或者是程序结束；

按F8：单步向下，不进入方法

F7：步入，单步向下，遇到用户自己写的方法，会进入执行

2、断点的类型#

IDE 中通常会提供四种类型的断点：

行断点：可作用在任何一行代码上，图标为红色圆形。当程序执行到设置断点的行时将会被触发。 方法断点：当断点被标记在方法的定义上时生效，图标为红色菱形，在方法的进入和退出时将被触发。（方法断点需要在断点处右键，勾选功能才能生效）

变量断点：只能作用在变量的定义上，图标为红色眼睛。当程序读取或者写入被标记的变量时，断点均会触发。 异常断点：当程序抛出相对应的异常时将会触发，作用范围为全局，图标为红色闪电。

3、断点的状态#

IDE中的断点状态通常分为八种，分别对应不同的情景。

常规状态：默认（原始）状态。 禁用状态：断点暂时处于禁用状态，不会被执行。 已验证状态：编译器会检查断点的设置是否合理。如果是，则将断点标记为已验证。 静音状态：断点暂时处于静音状态，不会被执行；其与禁用状态功能类似，不同点在于使用场景。 依赖状态：当一个断点的触发依赖于另一个断点时，显示为依赖状态。 挂起状态：挂起状态有两种类型，第一种是挂起全部线程，第二种是只挂起当前线程。当断点状态设置成了第二种方式后，只有当前线程会被堵塞，其他线程（程序）会正常执行，这在Spring Boot程序的测试中十分有用。 无效状态：断点的设置不是合理的，被标记的代码永远不可能被执行到。 警告状态： 如果断点的设置是合理的，但是存在其他问题，则编译器会向您发出警告。例如当被标记的方法可能不会被执行到时。 ———————————————— 版权声明：本文为CSDN博主「ThinkStu」的原创文章，遵循CC 4.0 BY-SA版权协议，转载请附上原文出处链接及本声明。 原文链接：https://blog.csdn.net/qq_35760825/article/details/125542220

二、Java的不同点#

int值和布尔值(boolean)之间不能相互转换

声明和定义没有区别

可以不用在数学方法和常量前面加上前缀“Math”，只用在源文件的最上面加上：

import static java.long.Math*

【编译过程】#

一个源文件（.java)只能有一个公共类，但可以有任意数量的非公共类，文件名必须与public类的名字相同

在编译源代码的过程中，编译器会在目录中创建类文件(.class)

启动程序，为字节码解释器提供陈序中包含main方法的那个类的类名

在多个源文件编译的过程中，一种是使用通配符调用Java编译器：

javac Employee*.java

另一种方法：

javac EmployeeTest.java

第二种方法没有显式的编译Employee.java，不过当Java编译器发现Employee.java中使用了Employee类时，他会首先查找Employee.class，如果没有发现，则会查找Employee.java并编译这个文件。另外，当Employee.java的版本较已有的Employee.class文件版本更新，Java编译器会自动地重新编译这个文件。

记录record#

record是一种特殊形式的类（class），其状态不可变，而且公共可读

record Point(double x,double y){ }

其结果有以下实例字段（被称为 组件）：

private final double x;

private final double y;

有一个构造器：Point(double x,double y)

以下访问器方法：

public double x()

public double y()

可以为记录添加自己的方法，可以有静态字段和方法

到那时不能为记录增加实例字段

对于完全由一组变量表示的不可变数据，要使用记录而不是类，如果数据是可变的，或者数据表示可能随时间改变则使用类。记录更易读更高效而且在并发程序中更安全。

【修饰符】#

final：

用关键字final指示常量，习惯上用全大写字母

final修饰符对于类型为基本类型或者不可变类型的字段尤其有用，但是对于可变类，使用final修饰符可能会造成混乱

static：

如果将一个字段定义为static，那么这个字段并不会出现在每个类的对象中。每个静态字段只有一个副本。

在一些面向对象的语言中，静态字段被称为类字段。

【数组】#

可以用变量初始化数组

长度为0的数组与null并不相同

创建一个数字数组时，所有元素都初始化为零，Boolean数组的元素会初始化为false。对象数组的元素则初始化为null。

【类】#

Java中的对象总是在堆中构造的，构造器总是结合new操作符一起使用

Employee number007(“James Bond”,10000,1950,1,1); //C++

Employee number007 = new Employee(“James Bond”,10000,1950,1,1); //Java

在Java中，所有的方法必须在类的内部定义，但是并不表示他们是内联方法，内联方法的设置时Java虚拟机的任务，对其进行优化。

Java程序设计语言总是采用按值调用。也就是说，方法会得到所有参数值的一个副本；具体来讲，方法不能修改传递给它的任何参数的变量内容。

不过有两种不同类型的方法参数：

1.基本数据类型（数字、布尔值）

2.对象引用

基本数据类型是不可能通过调用方法改变值的，但是对象引用却可以调用对象的方法进行修改。原因是：原来的对象引用和这个副本都引用同一个对象。

Java会自动完成垃圾回收，不需要人工回收内存，所以不支持析构函数。

快读#

static BufferedReader in = new BufferedReader(new InputstreamReader(System.in));
static Printwriter out = new Printwriter(new OutputStreamWriter(System.out));

利用Scanner类进行输入时正常情况下都是是够用。不过当输入量和输出量太大的时候（达到百万到千万），那么这种输入方式效率就很低。

StreamTokenizer#

这时候可以用StreamTokenizer类来输入基本数据类型，如 int double float boolean short long byte（只能这七种，其他不行，例如char，String）用法如下：

import java.io.*;

public class Main
{
    public static void main(String[] args) throws IOException
    {
      StreamTokenizer in = new StreamTokenizer(new BufferedReader(new InputStreamReader(System.in)));
      double a, b;
   while(in.nextToken() != StreamTokenizer.TT_EOF)
    {
       a = (double)in.nval;
       in.nextToken();
       b = (double)in.nval;
        System.out.println("a + b = "+(a+b));
    }
}

其中**in.nextToken（）**代表输入，遇到空格键，Enter键，Tab键结束一次输入。

（XXX)in.nval是将输入的数据转化成自己想要的数据类型(每输入一次转化一次），将in中的输入值传递给程序中的变量

【TIPS】nval读取到的结果是double类型的值,double类型通常有14~16位的精度,但是long类型的最大值会到9e18,所以有时就会造成精度丢失

BufferReader#

​ 用BuferrReader类（缓冲流）来输入的readLine()方法来输入字符串。当需要输入字符串次数超过1000次时，就应该用这个类输入，性能比Scanner类提升15ms左右。这个类主要有两个方法：read readLine.

**readLine():**输入一行的字符串，结束符为enter键。

**read():**返回从键盘输入的字符的SCAll值。（切记空格键，enter键也算字符）。

BufferReader主要是读取字符串，也可以用来读取基本类型只需要加上类型转换即可。

例如：

    static BufferedReader in= new BufferedReader(new InputStreamReader(System.in));
    static PrintWriter out =new PrintWriter(new OutputStreamWriter(System.out));
    public static void main(String[] args) throws IOException {
        long n=Long.parseLong(in.readLine());
        int m=Integer.parseInt(in.readLine());
    }
